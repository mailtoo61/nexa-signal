# TestFlight Execution Prep Guide

## Internal Build Prep

- Verify candidate build label and notes.
- Run smoke pass checklist before any external expansion.
- Confirm blocker board is current.

## External Build Prep

- Confirm external-wave readiness criteria are met.
- Ensure tester pack and reporting workflow are ready.
- Confirm known-issue communication draft is prepared.

## Tester Communication Prep

- Send welcome + test-focus guidance.
- Include reproduction template and escalation route.
- Set expectation for response windows.

## Build Labeling Discipline

- Maintain unique RC label per candidate.
- Keep build label, version, and commit mapping current.

## Smoke-Pass Requirements

- Fresh install
- Onboarding
- Gameplay loop
- Continue/recovery
- Settings persistence
- Legal/support paths

## Rollback Readiness

- Keep last known-good candidate available.
- Document rollback trigger criteria per wave.

## Issue Escalation Expectations

- Escalate release blockers immediately.
- Pause expansion on unresolved blocker.
- Log all escalation timestamps and owners.

## Tester Onboarding Flow

1. Invite accepted
2. Tester pack received
3. Baseline scenarios executed
4. Feedback submitted via approved template

## Known-Issue Communication Guidance

- Share only validated known issues.
- Avoid speculative or unverified claims.
- Mark workarounds clearly where applicable.
