# App Review Response Prep

## Common Reviewer Questions

- How does core gameplay progress work?
- Does the app require account creation or cloud sync?
- Where are privacy policy, terms, and support paths?
- Are accessibility controls available?

## Reduced-Motion Explanation

- Reduced motion is available in settings and user-controllable.
- Behavior changes are intended to reduce visual intensity while preserving function.

## Privacy / Support Clarification Notes

- App provides privacy and terms links in settings.
- Support path is available via support URL/email configured for release.
- Avoid legal/compliance claims beyond approved policy text.

## Resume/Recovery Clarification (If Asked)

- Resume/recovery is deterministic-safe and local-first.
- Invalid snapshot fallback starts fresh safely.
- UI does not fabricate gameplay authority.

## External Service Clarification

- Core gameplay does not require backend/cloud sync.
- Any future external services should be disclosed only when actually integrated.

## Reviewer Communication Tone

- Keep responses concise, factual, and respectful.
- Avoid defensive language.
- Provide exact reproduction path when clarifying behavior.

## Rejection Response Workflow

1. Capture rejection note verbatim.
2. Classify issue type (policy/quality/metadata/technical).
3. Assign owner placeholder (`[OWNER_TBD]`).
4. Draft response and remediation plan.
5. Verify fix and resubmit with clear reviewer notes.
